# DICE SLOT TRIP 開発仕様書

- 文書バージョン: v0.1
- 対象エンジン: Godot 4.7 stable
- 開発環境: Windows 11 / GDScript / Godot-MCP / Codex GPT-5.6 系
- 優先プラットフォーム: Android（縦持ち）
- PC版: 開発・デバッグ用として同時対応
- ゲーム種別: 1人用・世界旅行すごろく・収集・軽いスロット役

---

## 0. Codexへの依頼

この仕様書を設計の中心として、Godot-MCPを使い、**実際に起動して遊べる縦切りプロトタイプ**を段階的に完成させること。

細部に曖昧さがある場合は、毎回質問して停止せず、次の優先順位で妥当な判断を行う。

1. 触っていて気持ちよいこと
2. 休憩中に片手でぼんやり遊べること
3. サイコロを振るたびに小さな期待が生まれること
4. 将来ステージやボスをデータ追加しやすいこと
5. 実装の単純さと保守性

作業中は以下を徹底する。

- 各マイルストーンの終了時にプロジェクトを実行する
- GodotのDebugger、Output、警告を確認して修正する
- Godot-MCPで画面キャプチャを取得し、UI崩れや操作不能を確認する
- 一度に巨大なスクリプトへ詰め込まず、責務を分割する
- GDScriptは可能な範囲で静的型付けする
- 固定コンテンツをコードへ直書きせず、Resourceまたは外部データへ分離する
- 仮素材で先に遊べる状態を作り、ImageGenや素材生成Skillで段階的に差し替える
- 第三者アドオンは、本当に必要な場合のみ導入する
- 実装判断を行った場合は `docs/IMPLEMENTATION_NOTES.md` に短く記録する

# 1. ゲーム概要

## 1.1 タイトル

**DICE SLOT TRIP**

キャッチコピー候補:

> サイコロをそろえて、世界をめぐる。

## 1.2 コンセプト

世界各地を舞台にした巨大な一周ルートを、3個のサイコロで何度もぐるぐる周回する。

通常時は、サイコロを振って進むだけの穏やかな作業感を楽しむ。ときどきペア、ストレート、トリプルなどの役が成立し、ボスの気配、ワープ、レアイベントなどが発生して短い高揚が訪れる。

各ステージには、その土地を象徴する「ループボス」がいる。ボスを倒すのではなく、同じ個体と何度もエンカウントして交流し、交流ゲージを100%までためる。ゲージが満タンになるとその個体をゲットして図鑑へ登録し、次の個体が現れる。

## 1.3 プレイヤー体験

目指す感情の波は次のとおり。

> 静かな反復 → 小さな予兆 → 来るかもしれない期待 → 短い当たり → 柔らかな余韻 → 静かな反復

ゲームの8割は落ち着いていてよい。常時派手にせず、普段が静かだからこそ、役やボス遭遇がうれしく感じられる設計にする。

## 1.4 想定プレイ

- 1回: 20～30分程度を目標
- 縦持ちスマートフォン
- 片手操作
- オフラインで完結
- いつでも中断・再開可能
- 失敗して進行を失うゲームではない
- ボスゲージはセッションをまたいで保存される

# 2. 絶対に守るゲームの柱

1. **基本は3個の6面サイコロ**
2. **大きな一本道ループを何周もする**
3. **恒常的な分岐は増やしすぎない**
4. **止まったマスが毎周変わること自体を変化にする**
5. **何もないマスでループボスとランダムエンカウントする**
6. **同じ個体と繰り返し交流する**
7. **ボスは倒さず、交流ゲージ満タンでゲットする**
8. **ゲット後は同種の次の個体が出現する**
9. **周回するほど、そのセッション内で少し有利になる**
10. **色、BGM、SE、振動、演出は控えめにする**
11. **サイコロの役はスロット的だが、賭博や現金要素にはしない**
12. **複雑さは盤面の迷路ではなく、出目と出来事から生む**

# 3. 基本ゲームループ

1. タイトル画面
2. ステージ選択
3. キャラクター選択
4. 旅を開始
5. 基本3ダイスを振る
6. 出目の役を判定する
7. 合計値ぶんルートを進む
8. 止まったマスの効果を解決する
9. 通常マスならボスエンカウント抽選を行う
10. ボスに会った場合は短い交流を行い、ゲージを増やす
11. 周回地点を越えたら周回ボーナスを得る
12. ボスゲージが100%になったら個体をゲットする
13. 結果画面で図鑑登録と旅日記を確認する
14. 次のボス個体を生成して保存する
15. 次の旅へ

# 5. 盤面仕様

## 5.1 基本構造

- 1ステージはおよそ90マス
- ルートは閉じた単一ループ
- スタート地点はあるが、ゴールマスではない
- スタート地点を越えるたびに周回数を加算する
- 通常は時計回りの一方向移動
- 盤面全体を常時大きく表示せず、現在地付近を拡大表示する
- 画面上部などに簡易ミニマップを表示する
- 盤面背景は世界の観光地を眺めるジオラマ風にする

## 5.2 Godot上の実装案

- `Path2D` と閉じた `Curve2D` をルート本体として利用する
- ルート上にマス位置を等間隔または手動調整で配置する
- 各マスには連番の `tile_index` を持たせる
- プレイヤー位置は `tile_index` で管理する
- 移動時は次のマスへ順番にTweenする
- ミニマップは `tile_index / tile_count` の正規化位置から表示する
- ステージ固有背景とルートロジックを分離する

## 5.3 初期マス構成案

90マスを次の比率で仮配置する。遊びながら調整すること。

| 種類 | 数 | 役割 |
|---|---:|---|
| 通常マス | 50 | 小さな景色、ボス抽選、軽い報酬 |
| イベントマス | 14 | 1～2択の短い出来事 |
| アイテムマス | 8 | アイテム獲得 |
| コイン・お土産マス | 8 | 通貨または収集物 |
| ワープマス | 3 | 固定先または一定距離先へ移動 |
| ショップマス | 2 | 少数の商品を購入 |
| 休憩マス | 2 | 穏やかなボーナス |
| 名所マス | 3 | ステージ固有演出、ゲージ補助 |

合計: 90マス

## 5.4 マス効果の原則

- 原則として、止まったマスだけ発動する
- 通過マスは通常発動しない
- ワープ後は、移動先マスの効果を1回だけ発動する
- 同じイベントが短時間に連続しすぎないよう履歴を持つ
- マイナスイベントでも進行を巻き戻しすぎない
- 何も起きない通常マスにも、短い風景文や小さな内部進行を持たせる

# 6. サイコロとスロット役

## 6.1 通常ロール

- 基本は6面サイコロ3個
- 3個の合計値だけ進む
- 通常の移動期待値は10.5マス
- 役判定後に移動する
- ダイス演出は短く、通常時は約0.7～1.0秒を目安にする
- プレイヤー入力中以外は二重タップを受け付けない

## 6.2 役の分類

役は「メイン役」と「補助役」に分け、最大2つまで同時表示できるようにする。

### メイン役

| 役 | 条件 | MVP効果案 |
|---|---|---|
| PAIR | ちょうど2個が同じ | アイテムまたはお土産の追加抽選 |
| STRAIGHT | 1-2-3、2-3-4、3-4-5、4-5-6。順不同 | 移動後、近くの特別マスへ短い乗り継ぎが発生 |
| TRIPLE | 3個すべて同じ | 移動後にボス確定遭遇。交流量も少し増加 |

### 補助役

| 役 | 条件 | MVP効果案 |
|---|---|---|
| ALL ODD | 3個すべて奇数 | 次の報酬量とボス交流量が増えるが、イベント結果の振れ幅も増える |
| ALL EVEN | 3個すべて偶数 | 次の不利なイベントを軽減し、小さな安定報酬を得る |

## 6.3 役処理の優先順位

1. TRIPLE
2. STRAIGHT
3. PAIR
4. 補助役

メイン役は1種類のみ。補助役はメイン役へ重ねてもよい。

例:

- `3,3,5` → PAIR + ALL ODD
- `2,4,6` → ALL EVEN
- `5,5,5` → TRIPLE + ALL ODD

役の効果は今後調整可能なデータにする。判定ロジックへ報酬値を直書きしない。

## 6.4 1ダイスモード

仮名称: `ピンポイントダイス`

- サイコロを1個だけ振る
- 1～6マス進む
- スロット役は成立しない
- 止まったマスの基本効果を強化する
- 狙った近距離マスへ止まりたいときに使う
- アイテムまたはキャラクタースキルで発動する

## 6.5 5ダイスモード

仮名称: `トリップフィーバー`

- サイコロを5個振る
- プレイヤーは5個から有効にする3個を選ぶ
- 選んだ3個の合計値で移動する
- 選んだ3個だけで役を判定する
- 普段より少し考える、短い勝負どころにする
- 選択を面倒に感じる場合に備え、自動おすすめボタンを用意する

## 6.6 将来候補

MVP完成前には追加しない。

- 1個ホールドして残りを振り直す
- 特殊ダイス収集
- ステージ固有の役
- ダイス素材ごとのSE差分
- 出目を書き換えるレアアイテム

# 7. ターン進行

内部状態を明確なステートマシンで管理する。

```text
READY
  ↓
ROLLING
  ↓
ROLE_RESOLUTION
  ↓
MOVING
  ↓
LANDING
  ↓
EVENT_OR_ENCOUNTER
  ↓
TURN_RESULT
  ↓
AUTOSAVE
  ↓
READY
```

## 7.1 セッション長

初期値:

- 1セッションの基本ロール数: 36回
- 目標周回数: 約3～5周
- 目標プレイ時間: 20～30分

調整時は実測を優先する。

## 7.2 セッション終了条件

次のどちらかで結果画面へ進む。

1. 現在のボス個体をゲットした
2. 基本ロール数を使い切った

ボスをゲットできなかった場合も、交流ゲージと個体情報を保存する。失敗表示にはしない。

## 7.3 ボスゲット後

- 図鑑登録演出
- 今日の旅の簡易まとめ
- 次個体のシルエットまたは特徴を少しだけ表示
- 次個体を生成して保存
- 次のセッションでは新個体が登場する

MVPでは、1体ゲットした時点でそのセッションを終了する。

# 8. ループボス仕様

## 8.1 基本概念

各ステージには、現在交流中のボス個体が必ず1体いる。

例:

- 種族: スフィンクス
- 個体名: 眠そうなスフィンクス
- 性格: のんびり
- 好物: 冷たい飲み物
- 現在の交流ゲージ: 72%

同じ個体が何度も現れ、交流ゲージが100%になるまで関係が続く。

## 8.2 エンカウント

- 通常マスへ止まったときに抽選する
- 何度も外れ続けないよう、気配または天井を持つ
- プレイヤーには足跡などの5段階表示で気配を伝えてよい

初期調整値:

- 基本遭遇率: 12%
- 通常マスで外れるたび: +6%
- 最大遭遇率: 60%
- 遭遇後: 基本値付近へリセット
- TRIPLE時: 確定遭遇

数値はResourceまたは設定ファイルから変更可能にする。

## 8.3 交流ゲージ

- 最大値: 100
- 通常交流: 14～22程度
- 好物アイテム: 追加ボーナス
- 周回バフ、キャラクター能力、役で増減
- 目標: 1個体につきおよそ5～8回の遭遇

## 8.4 交流内容

交流は5～15秒程度を基本とする。

- 自動で進む短い交流
- 1～2択の簡単な選択
- 所持アイテムを渡す
- ステージ中に見たものを答える簡単な謎
- 写真を撮る
- 一緒に休む

不正解や相性違いでもゲージを完全に失わせない。

## 8.5 関係段階

| ゲージ | 反応例 |
|---:|---|
| 0～24 | 遠くから見ている。すぐ立ち去る |
| 25～49 | 近くに残る。短い反応を返す |
| 50～74 | プレイヤーを覚え、交流内容が増える |
| 75～99 | 特別な台詞やゲット予兆が増える |
| 100 | 心を開き、図鑑へ登録される |

## 8.6 カイロの初期個体案

MVPでは全個体を完成させなくてもよいが、データ上は追加可能にする。

- 眠そうなスフィンクス
- なぞなぞ好きのスフィンクス
- 答えを忘れたスフィンクス
- 子どものスフィンクス
- 観光客慣れしたスフィンクス
- 黄金のスフィンクス

個体差は次の組み合わせで作る。

- 色差分
- 表情
- 小物または装飾
- 性格タグ
- 好物タグ
- 交流台詞セット
- レア度
- 図鑑説明

同じ種族の再取得を、単なる素材変換にはしない。図鑑、生態メモ、見た目、台詞の収集を主な報酬にする。

# 9. 周回ボーナス

周回するほど、その旅の中で少し有利になる。

## 9.1 周回地点

- スタート地点を越えると周回数+1
- 周回数はゲーム画面上部へ常時表示
- 周回時は短いSEと控えめな演出を行う

## 9.2 周回ボーナス案

各周回時に、2～3個から1つだけ選ぶ。選択は数秒で理解できる内容にする。

- `足跡を追う`: ボス遭遇率を少し上げる
- `仲良し気分`: ボス交流ゲージ獲得量を少し上げる
- `旅慣れ`: アイテム・コイン報酬を少し上げる

効果はそのセッション中のみ累積する。

選択がゲームテンポを損なう場合は、キャラクターに応じた自動取得へ切り替えてよい。

## 9.3 恒久成長

MVP後の候補。

- ステージ熟練度
- 初期アイテム解放
- 個体出現テーブル追加
- 新キャラクター
- サイコロ外見
- 図鑑背景
- 旅日記の装飾

数値インフレより、選択肢と収集物が増える成長を優先する。

# 10. イベント仕様

## 10.1 原則

- 文章は短くする
- 選択肢は最大2つを基本とする
- 1イベントを数秒～15秒程度で終える
- 同じイベントの連続を避ける
- 土地らしさと小さなユーモアを優先する
- 不利な結果でも長時間を無駄にしない

## 10.2 カイロイベント例

### 砂の中で何かが光っている

- 掘ってみる: アイテム抽選
- 写真だけ撮る: ボス気配を増加

### 急な砂嵐

- 遺跡で休む: 次ターンの不利効果を無効化
- 走り抜ける: 成功するとコイン増加、失敗しても数マス戻るだけ

### 市場の冷たい飲み物

- 購入する: スフィンクスの好物を入手
- 眺める: 小さな旅日記を追加

### 石像の視線

- 見つめ返す: ボス気配上昇
- 手を振る: 低確率で個体固有イベント

## 10.3 イベントデータ

イベントは `EventData` として管理し、次の情報を持たせる。

- ID
- 表示名
- 本文
- 画像
- 選択肢
- 出現タグ
- 重み
- クールダウン
- 条件
- 結果リスト
- ステージID
- ボス個体タグ

複雑な任意コード実行をデータへ持たせず、定義済みの効果タイプを組み合わせる。

# 11. アイテム仕様

## 11.1 所持数

- セッション中の所持上限: 3～5個で調整
- 初期案: 4個
- 同一アイテムはスタック可能
- 使用確認は1タップで済ませる

## 11.2 MVPアイテム案

| アイテム | 効果 |
|---|---|
| ピンポイントチケット | 次のロールを1ダイスに変更 |
| トリップフィーバーチケット | 次のロールを5ダイスに変更 |
| ワープストーン | 数マス先、または登録済みワープ地点へ移動 |
| 冷たい飲み物 | スフィンクス交流時に追加ゲージ |
| 旅のお守り | 次の不利イベントを軽減 |
| おみやげ袋 | 次の報酬マスを強化 |

## 11.3 ショップ

MVPでは小規模にする。

- 商品は3件程度
- ゲーム内コインのみ
- 課金通貨なし
- 値引きや更新タイマーなし
- 画面遷移せず簡易ポップアップで購入可能

# 12. キャラクター仕様

MVPでは最低3人をデータ駆動で実装する。

## 12.1 のんびり旅人

- 初心者向け
- 1周につき1回、任意のダイス1個を振り直せる
- 全体的に安定

## 12.2 フォトグラファー

- 各周回で最初のボス遭遇時、交流ゲージに追加ボーナス
- 名所マスの報酬が少し増える

## 12.3 勝負師

- PAIR成立時、異なる1個を振り直してTRIPLEを狙える
- 振り直した場合、失敗するとPAIRの追加報酬を失う
- ときどき考えてドキドキしたい人向け

## 12.4 データ項目

- ID
- 表示名
- 説明
- 立ち絵
- 盤上トークン

# 14. UI・操作方針

- 論理解像度の基準: 720 x 1280、縦9:16
- 端末比率とセーフエリアへ対応する
- `Control`、`Container`、Anchorを使い、固定座標依存を減らす
- 最重要ボタンは親指が届きやすい画面下部へ置く
- 通常ターンは「サイコロを振る」の1タップで進められる
- 選択が必要なときだけ入力を増やす
- ダイス演出と移動演出はタップで短縮可能にする
- 文字だけでなく、形とアイコンでもマス種類を区別する
- 色だけを判別手段にしない
- 小さい文字を多用しない
- スマートフォン上で押せる領域を十分に取る

# 15. ビジュアル方針

## 15.1 全体

- 低彩度
- 柔らかいベージュ、砂色、淡い空色、くすんだ青緑
- 旅手帳、古い地図、絵本、ジオラマの中間
- かわいいが幼児向けには寄せすぎない
- 観光ポスターの原色感を避ける
- 派手なカジノUIにしない

## 15.2 通常時

- コントラストを抑える
- パーティクルを常時ばらまかない
- 背景はゆっくりした環境アニメーションだけ
- キャラクターは常時しゃべらない

## 15.3 予兆

- BGMへ音を1つ追加
- 遠景にボス影
- 足跡が増える
- ダイスの縁が淡く光る
- 背景の一部だけ変化する

## 15.4 当たり

- 強い白フラッシュを使わない
- 短く止める「間」を活用する
- TRIPLEでも演出を数秒以内にまとめる
- 当たり後の余韻を重視する

## 15.5 素材制作

- 初期は仮UI、単色アイコン、生成画像で進める
- 素材は差し替え可能な構造にする
- ImageGenでタイトル、背景、ボス、キャラクター、ステージカードを生成してよい
- UIアイコンは一貫した線幅と質感へ統一する
- 生成画像内の文字をそのままUIへ使用せず、GodotのControl上で文字を重ねる

推奨参照フォルダ:

```text
res://docs/reference/
```

ここへタイトル画面、ステージ選択、ゲーム画面のコンセプト画像を保存し、実装時の雰囲気の基準にする。

# 16. サウンド方針

## 16.1 BGM

- 曲として前へ出すぎず、環境音と薄い旋律の中間
- 長時間聞いて疲れないループ
- ステージごとに土地の空気を感じる音色
- 予兆時は別曲へ急に切り替えず、レイヤーを追加する

カイロ例:

- 遠い風
- 控えめな打楽器
- 小さな弦
- 市場のざわめき
- 砂の流れる音

## 16.2 SE

- ダイス: `ころ、ころ、こつ` 程度
- UI: 小さく柔らかい
- PAIR: 短い2音
- TRIPLE: 小さな3音と一瞬の無音
- ボス遭遇: 低く穏やかな合図
- ゲット: 長すぎない柔らかな余韻

## 16.3 振動

- デフォルトは弱め
- 通常着地: ほぼ感じない程度
- PAIR: 短く1回
- TRIPLE: 小さく3回
- ボス遭遇: ゆっくり1回
- 個別にオフ可能

# 24. やらない方向

- ログインを強制する
- ボス重複を数値素材へ変換するだけにする
- 周回を単なる数値作業にする
- コンテンツ追加のたびにゲーム本体へ個別コードを書く

## 最後に

このゲームで最も大切なのは、機能の多さではない。

> 何度もサイコロを振れる静けさと、たまに空気が変わる瞬間。

通常時を騒がしくせず、PAIRで少しうれしく、TRIPLEで短く息を止め、何度も出会ったボスが少しずつ心を開く。その手触りを最優先で実装する。
