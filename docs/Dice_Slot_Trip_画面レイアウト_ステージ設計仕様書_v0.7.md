# Dice Slot Trip
## 画面レイアウト・ステージ設計仕様書 v0.7

**更新日:** 2026-07-19  
**対象:** Godot 4.7 / Android 縦持ち 720×1280 / 片手操作  
**位置づけ:** `改善統合仕様書 v0.6` のうち、通常画面、カメラ、コース長、ステージ構成に関する内容を更新する追加仕様書。

---

# 0. 更新概要

本仕様書は、2026-07-19の家族フィードバックと手描き画面ラフを反映し、長時間の反復プレイに適した画面構成と、土地ごとに異なるステージ設計を定義する。

今回の重要な変更は以下。

1. **全ステージ共通の固定マス数を廃止する。**
2. **1周36マス前後を標準値としない。**
3. **通常時はキャラクターを画面内のほぼ同じ位置に置き、マス列を動かす。**
4. **常に進行方向の約6マスを見せる。**
5. **円環ワープ中のみ、円環全体を表示する。**
6. **サイコロ、スロット、キャラクターを近い位置にまとめ、視線移動を減らす。**
7. **文字、数値、アイコン、操作パーツを大きくする。**
8. **ステージの長さは、その土地で味わわせたい時間と構造から決める。**
9. **将来的に、短時間向けのタイムアタックステージを追加する。**

# 1. 画面設計の中心思想

## 1.1 目標体験

本作の通常画面は、地図全体を細かく観察する画面ではない。

> **近くの状況を大きく見せ、ぼんやり全体を眺めながら、何度もサイコロを振れる画面**

を目指す。

プレイヤーの主な視線対象は以下の3点。

1. キャラクター
2. 進行方向のマス
3. サイコロと3投スロット

この3点を画面中央から下部へ近接配置し、視線の往復距離を短くする。

## 1.2 優先順位

画面設計では、以下の順で優先する。

1. 視線移動の少なさ
2. マス種別の分かりやすさ
3. サイコロとキャラクターの近さ
4. 進行方向6マスの見通し
5. 大きな文字と操作パーツ
6. 長時間見ても疲れにくい動き
7. 土地ごとの旅情
8. 装飾性と豪華さ

装飾が視認性を損なう場合は、装飾を減らす。

# 2. 基本画面レイアウト

## 2.1 画面サイズ

- 基準解像度：720×1280
- 縦持ち固定
- 片手操作を前提
- 主要操作は画面下半分に集中
- 端末のノッチ、ステータスバー、安全領域を考慮する

## 2.2 画面の3領域

### A. 上部ステータス領域

画面高の約10〜13％を使用する。

表示候補：

- ステージ名または現在LAP
- HP
- 自己ベストとの差
- 現在地 / 総マス数
- 一時停止
- 全体ミニマップ表示ボタン

例：

`LAP 4 | HP ♥♥♡ | -2.4s | 18/54 | MAP`

要件：

- 数字は大きく表示する
- 主要数値を1行で読めるようにする
- 小さな説明文を置かない
- 頻繁に見る情報だけを置く
- タイム差はプラス・マイナス記号と数値で即読可能にする
- 色だけで状態を区別せず、形や記号も併用する

### B. 中央プレイ領域

画面高の約57〜63％を使用する。

配置：

- キャラクターは中央より少し下
- キャラクターの上方向に進行先を表示
- 通常時は進行方向約6マスを見せる
- 背景にはステージ固有の風景を置く
- マス、キャラクター、分岐表示を背景より優先する

要件：

- キャラクターを背景に埋もれさせない
- 先のマスが画面上端やUIに隠れない
- 次に止まれる可能性のある範囲が見える
- 分岐がある場合は、選択可能な方向を同時に見せる
- 背景の人物や装飾はマスと競合しない
- 重要情報を小さな文字だけで表現しない

### C. 下部ダイストレイ領域

画面高の約24〜30％を使用する。

配置候補：

- サイコロ
- READY / ROLL表示
- 3投スロット
- アイテムボタン
- 必要時のみ補助情報

推奨配置：

`[ DICE / READY ] [ SLOT 1 ][ SLOT 2 ][ SLOT 3 ]`

または縦幅に応じて、

```
[DICE] [SLOT 1][SLOT 2][SLOT 3]
[            READY            ]
```

要件：

- サイコロとスロットを近接させる
- キャラクターとの距離もできるだけ短くする
- スロット枠は常時見える
- 次に入る枠を明確にする
- 3枠目を過度に点滅させない
- 指で隠れても重要情報が失われない
- READY状態と目押し受付状態を明確に区別する

# 3. 通常移動画面

## 3.1 カメラ方式

通常移動中は、キャラクターを画面内のほぼ同じ位置に保つ。

- キャラクターはその場で小さく跳ねる
- マス列と背景が進行方向とは逆に動く
- プレイヤーにはキャラクターが旅を続けている感覚を与える
- 大きなカメラ移動や追従パンは行わない
- 視線の基準点を変えない

これは以前の「キャラクターが実際に画面内を前進し、カメラが追う」案を更新するものである。

## 3.2 進行方向6マス表示

通常時は、現在地から進行方向の約6マスを常に表示する。

目的：

- 出したい目を判断できる
- メリットマスとデメリットマスを見比べられる
- 次の3投スロットを考えられる
- ぼんやり見ても直近の状況を理解できる

表示ルール：

- 現在地を含めず、前方6マスを基本とする
- 地形や分岐によって5〜7マスの範囲は許容する
- マスの見かけサイズを極端に変えない
- 奥のマスほど小さくしすぎない
- 画面上端に近いマスも種別アイコンを判別できる大きさにする

## 3.3 マス移動演出

サイコロ結果が確定した後、1マスずつ進行を演出する。

- キャラクターはその場で小ジャンプする
- マス列が1マス分ずつ移動する
- 着地ごとに短い音または振動を入れられる
- 移動中も次のマス種別を追える
- 1マスあたりの演出時間は短くする

初期検証値：

- 1マス移動：0.14〜0.20秒
- 6マス移動：0.9〜1.3秒程度
- 高速設定ではさらに短縮可能
- 演出速度設定を将来的に用意する

## 3.4 分岐表示

分岐へ近づいた場合、通常の6マス表示内に分岐を見せる。

要件：

- 分岐の入口を大きな矢印で示す
- 分岐先の大まかなマス種別を2〜4マス分見せる
- 経路の長さや危険度を短い記号で示せる
- プレイヤーが選択するまで画面を大きく切り替えない
- 選択肢をキャラクター付近に置く

例：

- BYPASS
- SAFE
- COIN
- DANGER
- WARP

# 4. 円環ワープ画面

## 4.1 表示切り替え

円環へ進入した場合、通常の6マス先読み表示から、円環全体表示へ切り替える。

目的：

- 出口の位置を理解する
- 円環内のメリット・デメリット配置を比較する
- 欲しい出目を考える
- 何周しているか把握する

## 4.2 円環表示要件

- 円環全体を中央プレイ領域内に収める
- キャラクター位置を大きく表示する
- EXITを最も目立たせる
- EXITまで必要な出目を大きく表示する
- 全マスの大まかな種類を判別可能にする
- 通常画面とダイストレイ位置は変えない
- 円環表示へ切り替わっても視線基準を崩さない

表示例：

`EXIT 4`

## 4.3 円環内の判断

円環では以下が同時に見える必要がある。

- 出口までの距離
- 次のスロット完成候補
- メリットマス
- デメリットマス
- 現在位置
- 進行方向

出口へ早く戻るか、スロット役や報酬を優先するかを判断できる画面にする。

# 5. マスの視覚仕様

## 5.1 基本原則

各マスは、詳細な説明を読まなくても大まかな種類が分かるようにする。

識別には以下を併用する。

- 色
- アイコン
- 枠形状
- 短い記号
- 必要に応じた簡単なアニメーション

色だけには依存しない。

## 5.2 基本マス分類

### 通常マス
- 中立色
- 小さな足跡、道、点など
- 特別効果なし

### コインマス
- コインアイコン
- 円形または金属風の枠
- 黄・金系を基調とする

### 回復マス
- ハート、葉、しずくなど
- 緑系を基調とする

### 危険マス
- トゲ、警告、割れ目など
- 赤・橙系を基調とする

### ワープマス
- 渦、門、矢印など
- 青・紫系を基調とする

### 特殊マス
- 星、ダイス、鍵など
- 紫または固有色
- 効果ごとにアイコンを変える

### 分岐マス
- 分かれ道アイコン
- 大きな方向矢印
- 進行可能方向を明示する

### ボスマス
- 門、王冠、ボスシルエット
- 金属・赤・黒など
- 通常マスより大きくする

## 5.3 表示情報量

通常時は、マス名を常時文章で表示しない。

- 基本はアイコンで識別
- キャラクターが停止した時に短い名称を表示
- 詳細説明はタップまたは一時停止時に表示
- 先6マスを見ただけで、大まかな危険と報酬が分かることを優先する

# 6. 文字・パーツサイズ方針

## 6.1 基本方針

余白を広く取るより、情報と操作パーツを大きくする。

- 大きな数字
- 大きなアイコン
- 太い枠線
- 強い明暗差
- 短いラベル
- 少ない文章

## 6.2 最小タップ領域

- 基本：48×48dp以上
- 主要ボタン：56×56dp以上を推奨
- READYやROLLは片手で押しやすい大きさにする
- 小さなアイコンだけのボタンを乱用しない

## 6.3 可読性

- 日本語フォント：Noto Sans JP
- 数字は特に大きくする
- 背景上の文字には必ず十分なコントラストを確保する
- 装飾フォントはタイトルや短い見出しだけに使う
- 重要な状態を細字で表示しない

# 7. ステージ設計の基本方針

## 7.1 固定マス数の廃止

全ステージを同じマス数に統一しない。

ステージの長さは以下から決める。

- その土地で感じてほしい旅の長さ
- 地形や街路の特徴
- 分岐密度
- 円環の数
- イベント数
- ボス戦の長さ
- 想定プレイ時間
- 難易度曲線

> **現実の距離をそのままマス数へ変換するのではなく、その土地で感じてほしい時間をマス数へ変換する。**

## 7.2 ステージ選択時の表示

ステージカードには以下を表示する。

- ステージ名
- 想定プレイ時間
- マス数
- 難易度
- 分岐の多さ
- 円環の数
- 自己ベスト
- 主な特徴

例：

```
CAIRO GRAND ROUTE
目安 8〜10分
全58マス
分岐：多い
円環：2
BEST 07:42
```

## 7.3 想定時間の計算

ステージ時間はマス数だけで決めない。

概算式：

`想定時間 = 道中投射時間 + 移動演出 + 分岐判断 + 円環滞在 + イベント + ボス戦`

初期チューニングではプレイテストの中央値を採用する。

# 8. 通常旅行ステージ案

以下は初期検討値であり、実装時にプレイテストで調整する。

## 8.1 カイロ

- 仮マス数：54〜60
- 想定時間：8〜10分
- 構造：分岐多め
- 円環：1〜2
- 特徴：
  - 市場の細い道
  - 砂漠のバイパス
  - ピラミッドへ向かう終盤
  - 報酬と危険が混在
  - 長めの標準旅行ステージ

## 8.2 シンガポール

- 仮マス数：36〜44
- 想定時間：6〜8分
- 構造：コンパクト、高密度
- 円環：1〜2
- 特徴：
  - ワープや近道が多い
  - 分岐判断が頻繁
  - 短時間でも繰り返しやすい
  - 自己ベスト更新向け

## 8.3 モン・サン・ミシェル

- 仮マス数：44〜50
- 想定時間：7〜9分
- 構造：麓から頂上へ登る
- 円環：回廊や内部空間として配置
- 特徴：
  - 前半は穏やか
  - 後半ほど危険度が上昇
  - 頂上ボスへ向けて緊張が高まる
  - 一本道を基調に小さな脇道を持つ

## 8.4 ヴェネツィア

- 仮マス数：48〜56
- 想定時間：8〜10分
- 構造：橋と水路の分岐
- 円環：水路ループ
- 特徴：
  - 水路ワープ
  - 橋によるルート変更
  - 行き止まりに見える近道
  - ルート把握が重要

## 8.5 京都

- 仮マス数：42〜48
- 想定時間：7〜9分
- 構造：大通りと小路
- 円環：神社や庭園
- 特徴：
  - 比較的落ち着いたテンポ
  - 短い寄り道
  - 目押し精度を試す静かな区間

## 8.6 マチュピチュ

- 仮マス数：50〜58
- 想定時間：8〜10分
- 構造：段々道と高低差
- 円環：遺跡内部や段丘
- 特徴：
  - 危険なショートカット
  - 後半の上り区間
  - 高リスク高報酬の分岐

# 9. タイムアタック系ステージ

## 9.1 位置づけ

タイムアタックは、通常旅行ステージとは異なる独立モードとして将来実装する。

目的：

- 短時間で遊べる
- 自己ベスト更新に集中できる
- 同じコースを繰り返して熟練できる
- サイコロ目押し技術を明確に測れる

## 9.2 サーキット型

有名サーキットを想起させる周回型コースを使用する。

基本案：

- 2〜5周
- 同じコースを繰り返す
- ボス戦なし、または最終周のみ
- 分岐は少ない
- ゴースト表示あり
- ラップタイム記録
- セクタータイム記録
- コーナーと直線で狙う出目が変わる

例：

- 長い直線：大きい出目を狙う
- 急カーブ：特定の小さい出目でボーナス
- ピット：アイテム交換または安全策
- ショートカット：成功条件つき

## 9.3 グランツール型

複数ステージを連続して走る長期モード。

構成例：

- 平坦ステージ
- 山岳ステージ
- 個人タイムトライアル
- 長距離ステージ
- 最終ステージ

記録：

- 各ステージタイム
- 累計タイム
- 総投射回数
- 目押し成功率
- 区間別自己ベスト

## 9.4 権利と名称

実在の大会、サーキット、名称、ロゴ、コース形状を正式に使用する場合は、製品化前に権利確認を行う。

初期開発では以下を推奨する。

- 実在地点を参考にしたオリジナル名称
- ロゴや公式グラフィックを使用しない
- コース形状をゲーム向けに再構成する
- 土地の雰囲気と競技形式をモチーフとして扱う

# 10. 全体ミニマップ

## 10.1 役割

通常時は進行方向6マスのみ表示するため、全体構造を確認するためのミニマップ機能を用意する。

## 10.2 表示方法

- 上部のMAPボタンで表示
- ゲームを一時停止する
- 全体ルートを簡略表示
- 現在地、ゴール、分岐、円環のみ強調
- 全マスの詳細を見せる必要はない
- 閉じると元の視線位置へ戻る

ミニマップは常時表示しない。

# 11. ボス前後の画面

## 11.1 ボス前

通常の6マス表示を維持しながら、ボスゲートが徐々に大きく見える。

- 残り6マス以内でボスの存在を示す
- 背景の光、音、色を少しずつ変える
- UI配置は変えない
- 強いカメラズームは避ける

## 11.2 ボス戦

ボス戦でもサイコロと3投スロットの位置は維持する。

- キャラクターとボスを中央プレイ領域へ置く
- 通常時よりマップ表示を減らす
- HPと攻撃判定を大きくする
- 3投の履歴を常時見せる
- 操作位置を変えない

# 13. 検証項目

## 13.1 画面

- サイコロ、キャラクター、先6マスを一度に把握できるか
- 視線を大きく上下させずに遊べるか
- マス種別を文章なしで判別できるか
- 文字と数値が実機で十分大きいか
- 背景がマスやキャラクターを邪魔していないか
- 30分遊んでも視線疲労が強くないか

## 13.2 通常移動

- キャラクター固定でも前進感があるか
- マス列の動きが酔いにつながらないか
- 6マス先が判断に十分か
- 分岐が見切れないか
- 移動演出が長すぎないか

## 13.3 円環

- 円環全体を一度に理解できるか
- EXITまでの必要出目が即座に分かるか
- マス種類を見比べられるか
- 通常画面との切り替えで混乱しないか

## 13.4 ステージ長

- カイロが7〜10分程度の満足感を持つか
- シンガポールが短時間用として機能するか
- マス数より実プレイ時間が選択の目安になるか
- ステージごとの長さが個性として感じられるか
- 長さの違いが単なる水増しになっていないか

# 14. 受け入れ基準

最初の画面縦切りは、以下を満たした場合に合格とする。

1. 720×1280でHUD、6マス、キャラクター、トレイが同時に見える
2. キャラクターとサイコロの視線距離が短い
3. 6種類以上のマスをアイコンで判別できる
4. キャラクターが固定位置で跳ね、マスが1マスずつ移動する
5. 円環進入時に円全体が表示される
6. EXITまでの必要出目が大きく表示される
7. カイロのマス数を36に固定せず、データから変更できる
8. ステージ選択情報として想定時間とマス数を保持できる

# 15. 最終イメージ

通常時、プレイヤーは画面中央近くのキャラクターと、その先に並ぶ6マスを見る。

下にはサイコロと3投スロットがまとまっている。

プレイヤーがサイコロを止めると、キャラクターがその場で小さく跳ね、道が一マスずつ流れる。次に来るマスの色とアイコンが大きく見える。

ワープへ入ると、画面中央に円環全体が開く。出口まで4マス。コインマスと危険マスも同時に見える。

カイロは長く、シンガポールは短く密度が高い。モン・サン・ミシェルは頂上へ向かって少しずつ険しくなる。

プレイヤーはマス数ではなく、その日に遊べる時間と行きたい場所でステージを選ぶ。

> **世界の場所ごとに異なる長さとリズムを持ちながら、いつでも同じ視線と同じ指の位置で遊べる。**

これを本作の画面設計とステージ設計の中心とする。
