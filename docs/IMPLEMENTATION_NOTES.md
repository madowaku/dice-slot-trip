# Implementation Notes

- Godot 4.7 stable、Compatibility renderer、720 x 1280を基準にした。
- 盤面はPath2Dと閉じたCurve2Dを実行時に構築し、90地点を論理マスへ対応させる。
- M0〜M2では盤面全景を一画面に収め、M3以降で現在地周辺カメラとミニマップを分離する。
- STRAIGHTはM2では追い風の表示まで。特別マス自動乗り継ぎはM3の遭遇設計と同時に調整する。
- ワープは6マス先へ1回だけ移動し、移動先効果は連鎖させない。
- 5ダイスおすすめはTRIPLE、STRAIGHT、PAIR、補助役、合計値の順。
- 通常マスの一行旅メモ、5段階の気配、周回スタンプを低リスクな面白さ補強として追加した。
- 生成画像は背景・キャラクター・ボスをUI文字と分離して差し替える。
- 通常ロールは約1.3〜2秒で自動停止する。ロール中の追加タップで左から1個ずつ現在の目を早止めでき、集中すれば少し目押しを狙える。入力しなければ従来どおり1タップで完了する。
- 目押しは結果の保証ではなく短い技量余地とし、TRIPLEを恒常的に量産できない更新速度にする。
- ゲーム画面の中央は現在地を基準に約2.45倍で拡大し、移動の先読みと着地の手触りを優先する。全90マスのループは上部のミニマップで常時確認できる。
- M3では `マス種類と役割表v0.1` に合わせ、薄い灰青と足跡で識別できるボス気配マスを4つ追加した。停止時は気配を2段階上げ、次の通常マス抽選へ期待を持ち越す。
- `カイロ用イベント30本v0.1`、`基本アイテム20個 v0.1`、`初期キャラクター4人 v0.1` は今後のデータ駆動スライスの正式な入力として扱う。M3では交流ループの安定性を優先し、30イベント・20アイテム・4人固有能力の全実装は混在させない。
- キャラクター資料のうち「ボスゲットに必須のキャラクターを作らない」「交流進行はキャラクター変更後も維持する」はM3の保存設計と一致しており、現行の個体・図鑑状態は選択キャラクターから独立して保存する。
