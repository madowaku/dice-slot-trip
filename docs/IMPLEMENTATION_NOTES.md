# Implementation Notes

- Godot 4.7 stable、Compatibility renderer、720 x 1280を基準にした。
- 盤面はPath2Dと閉じたCurve2Dを実行時に構築し、90地点を論理マスへ対応させる。
- M0〜M2では盤面全景を一画面に収め、M3以降で現在地周辺カメラとミニマップを分離する。
- STRAIGHTはM2では追い風の表示まで。特別マス自動乗り継ぎはM3の遭遇設計と同時に調整する。
- ワープは6マス先へ1回だけ移動し、移動先効果は連鎖させない。
- 5ダイスおすすめはTRIPLE、STRAIGHT、PAIR、補助役、合計値の順。
- 通常マスの一行旅メモ、5段階の気配、周回スタンプを低リスクな面白さ補強として追加した。
- 生成画像は背景・キャラクター・ボスをUI文字と分離して差し替える。
- 通常ロールは約1.3〜2秒で自動停止する。ロール中の追加タップで左から1個ずつ現在の目を早止めでき、集中すれば少し目押しを狙える。入力しなければ従来どおり1タップで完了する。
- 目押しは結果の保証ではなく短い技量余地とし、TRIPLEを恒常的に量産できない更新速度にする。
- ゲーム画面の中央は現在地を基準に約2.45倍で拡大し、移動の先読みと着地の手触りを優先する。全90マスのループは上部のミニマップで常時確認できる。
- M3では `マス種類と役割表v0.1` に合わせ、薄い灰青と足跡で識別できるボス気配マスを4つ追加した。停止時は気配を2段階上げ、次の通常マス抽選へ期待を持ち越す。
- `カイロ用イベント30本v0.1`、`基本アイテム20個 v0.1`、`初期キャラクター4人 v0.1` は今後のデータ駆動スライスの正式な入力として扱う。M3では交流ループの安定性を優先し、30イベント・20アイテム・4人固有能力の全実装は混在させない。
- キャラクター資料のうち「ボスゲットに必須のキャラクターを作らない」「交流進行はキャラクター変更後も維持する」はM3の保存設計と一致しており、現行の個体・図鑑状態は選択キャラクターから独立して保存する。
- M4Aは `data/events/cairo_m4a_events.json` の代表10本を `EventSystem` が抽選・条件解決し、`RewardResolver` が状態へ適用する。イベント固有コードからコインや記録を直接変更しない。
- イベント開始時に到着ダイス、合計、役、早止め、キャラクターを複製する。追加ダイスは既存ロール演出と早止めを再利用し、到着情報と現在マスを変更しない。
- 通常イベントの役優先順位はJSONのoutcomes順（TRIPLE→STRAIGHT→PAIR→補助役→合計→基本）で表現する。CAI-E30のみ5個全体から成立役種類とFIVE_OF_A_KINDを集計する。
- セーブv3はイベント履歴、直近ID、周回レア制限、進行中イベント、適用済みresolution_id、旅メモ、ポストカード、予約効果を追加する。v1/v2では欠けた項目を既定値で補う。
- M4Aのアイテム候補は共通2候補の最小実装。基本アイテム20個の固有使用効果と初期キャラクター4人の固有能力は後続スライスで接続する。
- M4Aイベント報酬は数値仕様v0.1に合わせ、COIN_XS/S/M/L/JACKPOTを5/12/30/70/120、所持上限99,999とする。ITEM_COMMONはCOMMON確定、ITEM_CHOICEは最低1 COMMONかつ第2候補COMMON75%/UNCOMMON25%、ITEM_RAREはUNCOMMON70%/RARE30%。SKILL_RECOVERはcharacter_skill_chargeのみを1へ戻し、満タンならCOIN_S（12）へ変換する。
- T004では `_animate_dice_roll` を通常・イベント追加ダイスの共通入口のまま維持し、転がり中だけ軽い回転、拡縮、色変化を与えた。停止済みダイスは水平・等倍へ戻るため、目押し状態を視覚的に判別できる。
- 通常ロールは既存の主ボタンで左から1個停止し、補助ボタンで未停止分を現在値のまま一括停止する。追加ロールではモーダル内へ同じ2操作を出す。停止済み個数を単一カウンタで管理し、一括停止後の再入力は無視する。
- T005では90マス境界を通常移動またはWARPで越えた時に、E30の周回内使用済み状態・周回イベント履歴・レア後カウンタを初期化する。
- イベントからのボス交流予約は結果画面を閉じても保存上維持し、実際のボス交流モーダル生成後に一度だけ消費して再保存する。これにより画面間で終了しても次回起動時に交流へ戻り、消費後の再起動では重複しない。
- CAI-E30の高Tier報酬は `RARE_EVENT` プールとして通常ITEM_RAREから分離し、UNCOMMON 55% / RARE 45%を使用する。
- M4AのQAキャプチャはviewportを360×640へ正規化し、背景色へアルファ合成後にRGB PNGとして保存する。これによりウィンドウ合成由来の透明領域や黒帯が証跡へ混入しない。共通本文 `_body()` も既定でINK色を指定する。
- セーブv5では恒久解禁の`unlocked_dice_count`を一時状態`current_dice_count`へ置換した。旅開始は1、2ダイスDOUBLEで3、3ダイス後はTRIPLE=3、PAIR/STRAIGHT/ALL EVENのみ=2、それ以外=1へ遷移する。ロール後の消費を着地報酬より先に行い、追加ダイス報酬の順序を仕様どおり固定する。
- `DICE_ADD_1`、`DICE_SLOT_READY`、`DICE_KEEP`を共通報酬として解決し、既存イベントの`DICE_UNLOCK`は`DICE_ADD_1`の互換aliasとして残す。3個を超える追加ダイスと重複KEEPはCOIN_S（12）へ変換する。
- v4の恒久3ダイス状態はv5読込時にDOUBLE CHANCE（2ダイス）へ一度だけ移行する。イベント専用の追加1/3/5ダイスは到着時の`current_dice_count`を変更せず、通常UIからの手動ダイス数変更も無効化した。
- 改訂盤面v0.2は5地区×18マスの90要素を`BoardModel.CAIRO_TILES`へ明示した。NORMAL 39、EVENT 11、ITEM 10、COIN 6、WARP 3、SHOP 3、REST 4、LANDMARK 3、BOSS_SCENT 4、STAGE_SPECIAL 2、RISK 5。RISK循環間隔は17/14/10/12/37、ITEM最小4、重処理マス最小4をテストで固定する。
- ITEMマスは0〜34=DICE_ADD_1、35〜89=通常アイテム、90〜99=2候補の純粋境界関数を使用する。DOUBLE着地は追加ダイス進行を重複overflowとして扱わず、通常アイテムを必ず併給する。通常の3ダイス余剰は従来どおりCOIN_Sへ変換する。
- RISKマスは安全策（追加ダイス+1）と挑戦（移動なし追加3ダイス）の短いモーダル。挑戦報酬はTRIPLE/STRAIGHT/PAIR/ALL EVEN/ALL ODD/役なしで分岐し、どの結果でも到着時の`current_dice_count`を下回らない。盤面では赤い警告マーク、STAGE_SPECIALは青緑の十字光で拡大画面とミニマップの双方から判別できる。
- 2.5Dダイスは640×280透明SubViewport、Perspective FOV 30、暖色Keyと弱い寒色Fillで構築した。最大5個を起動時に生成し、各ダイスはBoxMeshと6面21pipのCylinderMesh MultiMeshを持つ。ロールごとのノード生成は行わない。
- 出目は物理姿勢から取得せず、既存ロジック値を`orientation_for_face()`へ渡す。ローカル面は上/下=1/6、前/後=2/5、右/左=3/4で、反対面関係と全6姿勢をpure testで固定した。
- 表示状態はREADY/ROLLING/SETTLING/LOCKED。既存`locked_dice_count`を同期元として、次に止める1個だけ弱く金色highlightし、ロール中は小さなlaunch/bounce、停止時は0.18秒のsettleを行う。自動ロールは約0.8〜1.3秒へ調整し、早止め・一括停止・固定出目・追加1/3/5・RISK追加3ダイスで同じpresentationを再利用する。
- 5ダイス選択時だけ既存2D出目ボタンを操作overlayとして表示し、それ以外は3D表示を使用する。3D scriptが生成できない場合のfallbackとして2D描画コード自体は維持する。
- プレミアムUIでは背景artの表示率を上げつつ、文字情報を金縁parchment／砂革panelへ集約した。上部は周回・コインpill、ボスportrait＋実ProgressBar、独立90マスminimapの順で一目の階層を作る。
- 現在地拡大は全曲線の単純zoomを廃止し、現在±5の11マスを番号・特殊色・記号・NEXT付きの連続タイルリボンとして描画する。プレイヤーを中央固定し、RISKは赤色と`!`で停止前から判別できる。
- ダイス領域は丸い砂革panelと金縁へ変更し、3D floorを低彩度の革色へ調整した。ダイス表示は通常約1.24倍、1ダイス時約1.48倍。旧mode Buttonは廃止し、NORMAL / DOUBLE CHANCE / DICE SLOTの非操作progress indicatorへ置換した。
- 360×640のone/two/three/risk/rolling captureを参考画面と比較し、縦組み、暗すぎるtray、黄味過多plane、小さいplayerをP0/P1として修正した。比較監査とP3候補は`design-qa.md`へ記録する。
