\# Dice Slot Trip



\## カイロ用イベント30本 v0.1



\### 1. イベント設計方針



カイロのイベントは、プレイヤーの旅を中断する大事件ではなく、移動の途中に挟まる短い出来事として設計する。



基本方針は以下。



\* 1イベントは5〜15秒程度で終わる

\* 文章は原則1〜2行

\* 到着時のサイコロ結果を再利用する

\* 追加ダイスは珍しいイベントだけで使用する

\* 失敗しても大きく後退しない

\* 戦闘、HP減少、アイテム全消失は使用しない

\* 同じイベントでも役によって小さな差が出る

\* 初回と再発生時で文章を変えられる構造にする

\* 世界観説明よりも、旅先で何かを見つけた感覚を優先する



\---



\# 2. 仮報酬記号



数値バランスを後から変更しやすいよう、イベントデータでは以下の記号を使用する。



| 記号             | 内容             |

| -------------- | -------------- |

| COIN\_S         | 少量のコイン         |

| COIN\_M         | 中量のコイン         |

| COIN\_L         | 多量のコイン         |

| ITEM\_COMMON    | 通常アイテムを1個      |

| ITEM\_CHOICE    | 2候補からアイテムを1個選択 |

| ITEM\_RARE      | レアアイテム抽選       |

| BOSS\_SCENT\_1   | ボスの気配を1段階上昇    |

| BOSS\_SCENT\_2   | ボスの気配を2段階上昇    |

| INTERACT\_BONUS | 次回交流ゲージに小ボーナス  |

| SKILL\_RECOVER  | 回数制スキルを1回復     |

| STAMP\_NOTE     | 旅メモまたは図鑑メモを登録  |

| POSTCARD       | 特別なポストカードを登録   |

\# 9. 抽選ルール



\## 地区ごとのイベントプール



現在地の地区に対応する6本を基本プールとする。



例：



\* 市場地区：CAI-E01〜E06

\* ピラミッド地区：CAI-E07〜E12

\* オアシス地区：CAI-E13〜E18

\* 遺跡地区：CAI-E19〜E24

\* 砂丘地区：CAI-E25〜E30



10〜20%程度の確率で隣接地区のイベントを混ぜてもよい。



\## 再発生制御



\* 直前に発生したイベントは再抽選しない

\* 直近3件のイベントは抽選率を下げる

\* 初回未発生イベントには抽選補正を付ける

\* レアイベントCAI-E30は連続発生させない

\* 同一周回内でCAI-E30は最大1回



\## 初回と再発生



各イベントは以下を保持できる構造にする。



\* 初回文章

\* 再発生文章

\* キャラクター固有文章

\* ボス個体固有文章

\* 時間帯固有文章



v0.1では初回文章だけでも動作可能とする。

\# 10. 役処理の優先順位



複数の役が同時成立した場合、イベント側では以下の順で特殊反応を判定する。



1\. TRIPLE

2\. STRAIGHT

3\. PAIR

4\. ALL ODD／ALL EVEN

5\. 合計値条件

6\. 基本結果



5ダイスでは複数役を同時処理できるイベントも存在する。



通常イベントでは報酬が過剰にならないよう、原則として最も優先度の高い専用効果一つだけを適用する。
