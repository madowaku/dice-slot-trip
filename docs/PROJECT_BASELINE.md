## 3. 固定したゲームの核

- 1投1サイコロ。プレイヤーが出目を狙って止める
- 出目は即座に移動へ使い、同時に3枠のスロットへ左から保存する
- 3投ごとに `TRIPLE > STRAIGHT > PAIR > MIX` の優先順で判定する
- 報酬はスキルゲージとコインへ絞る
- 前方6マスとスロット履歴の競合を主な判断にする
- ルートは本線（安全）、バイパス（速いが危険）、円環（遅いが報酬と精密停止）を使い分ける
- 探検猫のピンポイントは次の1投を見やすくするが、出目を書き換えない
- 各周の最後は13マスの反転ダイス・ボスレース。ボス出目は `7 - 有効なプレイヤー出目`
- ボス戦は勝敗にかかわらず記録して完走し、リトライしない

## 4. カイロ製品スライス

- 本線58マス、4地区
- バイパス2本
- 円環2か所
- 1周8〜10分
- ボスレース25〜45秒
- 通常時は昼、ボスレースは夜

## 5. 現行実装

| 領域 | v1ランタイム |
|---|---|
| 起動導線 | `V1PlayScreen.tscn` を既定起動に設定 |
| 3ROLL | `TRIPLE > STRAIGHT > PAIR > MIX`、ゲージと超過コインを実装 |
| スキル | ピンポイントのREADY/ARMED/ACTIVE/LOCKEDを実装 |
| コース | YAMLから生成した58マス本線・分岐・円環・ワープ・ボスゲートを実装 |
| ボス | 13マス並走、反転ダイス、最大12ターンの勝敗記録を実装 |
| UI | セッション状態だけで動く独立v1プレイ画面を実装 |
| アート | v1ランタイムは組み込みControl中心。新規素材は `assets/art/cairo/v1/` へ追加 |

旧コードは既定起動から外れており、仕様の根拠にはしない。未追跡の旧アートや制作元は、別トランシェで参照と権利情報を監査してから整理する。

## 6. 実装と検証の状態

- YAML subset parserとbyte-exact生成物チェックを実装済み
- 3ROLL SLOT、スキル、プレイセッション、コース移動、ボスレースをテスト済み
- v1専用UIと既定起動への切替を実装済み
- V06専用runtime/data/testsを参照ゼロ確認後に削除済み
- 旧アート全体の監査と広範なlegacy cleanupは後続トランシェへ延期

## 7. 削除判断

今回削除してよいものは、再生成可能なキャッシュ・ビルド、作業動画、旧QA証跡、完了済みGoal Makerログ、明白な差分ファイルと複製ファイル。

旧コード・旧データ・旧アートは、参照確認なしで追加削除しない。既存dirtyな旧画面系コードと未追跡素材は、今回のv1公開範囲へ混ぜずローカルに保全する。
