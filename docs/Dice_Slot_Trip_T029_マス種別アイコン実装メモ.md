# Dice Slot Trip
## T029 マス種別アイコン実装メモ

**目的:**  
NORMAL / COIN / REST / RISK / ITEM / EVENT などのマスを、色だけに頼らず、遠目でも形とアイコンで判別できるようにする。

**対象:**  
Godot 4.7 / Android縦持ち 720×1280・360×640 / 通常昼マップ・ボス夜景

---

## 1. 実装の中心方針

各マスは、以下の3層で識別する。

1. **外形**
2. **中央アイコン**
3. **補助色**

色を見なくても大分類が分かり、近くでは中央アイコンで意味を確認できること。

---

## 2. 種別ごとの推奨ルール

| 種別 | 外形 | 中央アイコン | 補助色 | 印象 |
|---|---|---|---|---|
| NORMAL | 丸みのある四角 | 足跡 / 小さな菱形 | 砂色・中立色 | 安全な足場 |
| COIN | 円形 / メダル枠 | コイン / 星 | 黄・金 | 報酬 |
| REST | 葉形 / 柔らかい角丸 | ハート / 葉 / しずく | 緑 | 回復・休息 |
| RISK | ギザ枠 / 警告三角 | トゲ / ひび / 警告 | 赤・橙 | 危険 |
| ITEM | 箱形 / 荷物枠 | バッグ / 宝箱 / 道具 | 青 | 入手 |
| EVENT | 六角 / 特殊枠 | ？ / 巻物 / 星 | 紫 | 予測不能 |
| WARP | 円環 / 渦巻き枠 | 渦 / 門 | 青紫 | 転移 |
| BOSS | 大型ゲート形 | 王冠 / ボス紋章 | 金・黒・赤 | 山場 |

---

## 3. アイコン制作ルール

- 1マスにつき中央アイコンは1つ
- 細線を避け、太いシルエットで作る
- 360×640でも潰れない
- 文字は基本使わない
- 小さな装飾は省略する
- 立体マスと遠景の平面マスで同じ記号体系を使う
- アイコンの向きや明暗を統一する
- 昼夜で別デザインにせず、同一記号を使う

---

## 4. NORMALの扱い

NORMALは「無情報」にしない。

- 安全な足場として見える
- 特殊マスより弱いが、背景に消えない
- 小さな足跡、道標、菱形などを使う
- 連続した時に道のリズムが整って見える

---

## 5. 表示優先度

通常プレイ中、先6マスを以下の順で読み取れること。

1. RISK
2. REST
3. COIN
4. WARP
5. ITEM / EVENT
6. NORMAL

RISKとRESTは最も区別しやすくする。

---

## 6. 昼夜両対応

以下の4条件で確認する。

- 昼背景＋平面マス
- 昼背景＋現在地周辺の立体マス
- 夜背景＋平面マス
- 夜背景＋ボス付近の立体マス

要件：

- アイコンが背景に埋もれない
- 白飛び・黒潰れしない
- 発光表現に頼らなくても判別できる
- 夜景では輪郭線または内側プレートで視認性を保つ

---

## 7. 実装構造案

各マス種別をデータ駆動にする。

```text
TileVisualSpec
- type
- shape_id
- icon_texture
- base_color
- border_style
- pulse_style
- daylight_variant
- night_variant
```

ただし daylight / night は色・輪郭補正のみとし、記号体系そのものは変えない。

---

## 8. 最小実装スコープ

最初のスライスでは以下を実装する。

- NORMAL
- COIN
- REST
- RISK
- ITEM
- EVENT

可能なら同時にWARPも追加する。

---

## 9. 受け入れ基準

1. 色を見なくても6種を形で区別できる
2. 前方6マス表示で危険と回復を1秒以内に見分けられる
3. 720×1280と360×640の両方でアイコンが潰れない
4. 昼マップと夜ボス画面の両方で視認できる
5. 立体マスと平面マスで意味が一致する
6. 背景やキャラクターより強く主張しすぎない
7. 既存の前方6マス表示、MAP overlay、ボス画面を壊さない
8. 既存テスト、専用テスト、資産検証をすべて通す

---

## 10. 推奨検証キャプチャ

同じ進行状態で以下を取得する。

- 通常昼 720×1280
- 通常昼 360×640
- ボス夜 720×1280
- ボス夜 360×640
- 6種が同時に前方へ並ぶテスト画面
- 色を落としたグレースケール確認

---
