# Dice Slot Trip
## Roll-to-Move Feel Pass 実装計画書 v0.2

**作成日:** 2026-07-20  
**対象:** Godot 4.7 / Android縦持ち / 360×640基準・720×1280検証  
**対象画面:** 通常観光マップ、ボス前後のマップ表示、3ROLL SLOT  
**目的:** サイコロ停止からキャラクター移動までの一投体験を、見やすく、気持ちよく、長時間遊んでも疲れにくいものへ改善する。

---

## 0. この文書の使い方

この文書は、既存コードを調査したうえで実装するための計画書である。  
既存の責務・セーブ・MAP・ボス・アイテム／スキルドック・3ROLL SLOTを維持し、今回のスライスに必要な範囲だけを変更する。

### 0.1 先に確認できた既存実装

現時点のコードには、次の要素がすでに存在する。したがって、今回の作業は「ゼロから導入」ではなく、既存実装の整理・固定・視認性改善として進める。

- scripts/app/main.gd
  - TurnPhase の一部と、rolling_dice / moving の互換用フラグ
  - SETTLING → RESULT_LOCK → SLOT_TRANSFER → MOVE_PREP → MOVING の待ち時間
  - ダイス停止、ロールトランザクション、移動・マス効果への接続
- scripts/game/map_dice_overlay.gd
  - MapDiceOverlay.Phase
  - マップ上への投射、早止め、結果保持、トレイへの復帰
  - MapDieBillboard、スロット枠、ピクトグラム描画との連携
- scripts/game/tourism_map_view.gd
  - FIRST_OFFSET = -1、LAST_OFFSET = 6、SLOT_COUNT = 8 相当の近傍表示
  - 遠方まで続くルートポリライン
  - 前方オフセット、着地点表示、分岐描画、円環系ルートの既存分岐
- scripts/game/v06_play_session.gd
  - ゲーム進行用の READY / MOVING / CHOICE_REQUIRED / RESOLUTION_REQUIRED など
  - これは今回のUI演出用 TurnPhase とは別のドメイン状態である

### 0.2 重要な設計判断

1. main.gd の TurnPhase は一投のUI／入力シーケンスを管理する。
2. MapDiceOverlay.Phase はダイス表示の内部状態を管理する。
3. V06PlaySession のフェーズはゲームルールを管理し、今回のUIフェーズで置き換えない。
4. rolling_dice と moving は移行期間の互換プロパティとし、新規処理から直接状態遷移に使わない。
5. turn_phase を唯一の入力判定元にし、互換フラグから turn_phase を勝手に巻き戻さない。
6. 画面描画の確認を自動テストの合格条件に含めるが、画像・動画でしか判断できない項目は手動検証として分離する。

## 1. 現在の問題

### 1.1 サイコロ

- 台座の側面が台形に見え、トレイ上ではなく別の台に乗っているように見える
- 空中で回転している印象が強く、接地感・重量感が弱い
- 回転、停止、着地のつながりが急である
- 毎フレームの変化が均一で、玩具らしい規則性と揺らぎのバランスが弱い
- 影がダイス本体の回転や移動表現と混ざり、浮遊感が残る

### 1.2 出目から移動まで

- 停止直後に結果処理とキャラクター移動が近接し、出目を見る時間が短い
- サイコロ、スロット、キャラクター、マップへ視線が同時に要求される
- 停止後の処理が複数のフラグと await に分散すると、二重入力や古いシーンへの遅延処理が起きやすい

### 1.3 マップ

- 近傍以外のタイルが操作対象のように見える
- 画面内のタイル数が多いと、各タイルのアイコンが小さくなる
- 現在地との差を絶対マス番号から暗算する必要がある
- 分岐・円環ルートでは通常の直線表示だけでは判断材料が不足する

### 1.4 マス種別

- 足跡や抽象記号だけではNORMAL、移動効果、BOSS系を区別しにくい
- 文字・色だけに依存すると、小画面・多言語・夜景で判読性が落ちる
- board_view.gd と tourism_map_view.gd で記号体系が分かれると、同じ種別の意味が揺れる

## 2. ゴール

1. ダイスがトレイ上で軽く跳ね、滑らかに減速して接地する。
2. 停止入力時に各ダイスの論理出目と表示面を一度だけ確定する。
3. 確定後の SETTLING 中に、出目・ピップ・表示面が変化しない。
4. ダイス停止からキャラクター移動開始まで、500〜650ms程度の読解時間を確保する。
5. 通常の直線マップは「直前1＋現在地＋前方6」の8スロットで表示する。
6. 前方6マスに相対出目 1〜6 を表示し、現在地だけ絶対番号を表示する。
7. 8スロットの外側はタイルではなくルート線で表現する。
8. タイルの外形・ピクトグラム・補助色の3層で種別を判別できる。
9. 分岐では本線と近道の比較を補助表示し、円環ではEXIT距離を最優先する。
10. pause/resume、MAP、ボス遷移、シーン終了時に、途中の演出が別の状態へ作用しない。

## 3. 非ゴール

- カイロの全マス配置、ステージデータ、報酬量
- 新規アイテム、スキル、ボスルール
- PAIR / TRIPLEの成立条件・報酬設計
- ステージ選択画面、全体UIの再デザイン
- ダイスを完全物理シミュレーションへ置換すること
- V06PlaySession のゲームルール用フェーズをUI演出用フェーズへ統合すること
- 既存のセーブ形式、ロールトランザクションの意味を変更すること

## 4. フェーズと責務

### 4.1 UI／ターンフェーズ

main.gd の TurnPhase を一投の入力・演出シーケンスの正とする。

    enum TurnPhase {
        READY,
        ROLLING,
        SETTLING,
        RESULT_LOCK,
        SLOT_TRANSFER,
        MOVE_PREP,
        MOVING,
        RESOLVING_TILE,
        PAUSED
    }

PAUSED に遷移する前の状態は、別変数 phase_before_pause に保存する。  
PAUSED から復帰するときは常に、保存した同一フェーズへ戻す。

### 4.2 フェーズ遷移

    READY
      └─ ロール開始
         ↓
    ROLLING
      └─ 各ダイスの停止入力／自動停止
         ↓
    SETTLING
      └─ 全ダイスの表示面固定、着地演出
         ↓ 140ms
    RESULT_LOCK
      └─ 出目の読み取り、着地点の予告
         ↓ 200ms
    SLOT_TRANSFER
      └─ 3ROLL SLOTまたは履歴枠へ反映
         ↓ 140ms
    MOVE_PREP
      └─ キャラクターの沈み込み・移動予告
         ↓ 100ms
    MOVING
      └─ 1マスずつ移動
         ↓
    RESOLVING_TILE
      └─ マス効果、分岐、円環、ボス遷移
         ↓
    READY

PAUSED は上記のどの非終端フェーズからも一時退避できる。  
ただし、シーン終了・ロールキャンセル・エラー時は READY または既存の安全な終了状態へ明示的に遷移し、PAUSED を残さない。

### 4.3 フェーズと既存クラスの対応

| 責務 | 正となる状態 | 変更方針 |
|---|---|---|
| ロール受付、入力ロック、待ち時間 | main.gd: TurnPhase | 今回の中心。遷移を一元化する |
| ダイスの投射・早止め・保持・復帰 | MapDiceOverlay.Phase | TurnPhase と二重管理しない。公開APIを通して同期する |
| ルート、移動、マス効果、ボス | V06PlaySession または既存のGameState | UIフェーズで置き換えない |
| タイルの見た目 | tourism_map_view.gd / 共通仕様 | 種別マッピングを共有化する |

### 4.4 互換フラグ

既存コードとの互換のため rolling_dice と moving を残す場合も、次の制約を守る。

- 新規処理は turn_phase を参照する
- 互換フラグは turn_phase から導出する
- 互換フラグのsetterから、別のフェーズへ暗黙に戻さない
- 移行完了時に直接書き換えている箇所を検索し、段階的に削除する

## 5. 入力契約

    func can_start_roll() -> bool:
        return turn_phase == TurnPhase.READY and not modal_open

    func can_stop_roll() -> bool:
        return turn_phase == TurnPhase.ROLLING

    func can_open_map() -> bool:
        return turn_phase in [TurnPhase.READY, TurnPhase.RESOLVING_TILE] and not modal_open

    func can_open_item_dock() -> bool:
        return turn_phase == TurnPhase.READY and not modal_open

### 必須条件

- SETTLING 以降の停止入力は無視する
- 同じダイスの停止イベントを二重に受け付けない
- SLOT_TRANSFER を二重実行しない
- MOVING 中に新しいロールを開始しない
- ROLLING 中にMAPを開かない
- MAPを開いている間は、待ち時間・演出タイマーを進めない
- pause中は、待ち時間・Tween・移動ステップを進めない
- resume後は phase_before_pause の同じ処理位置から再開する
- シーン終了後に、古いシーケンスの await が新しいシーンへ作用しない

### シーケンス無効化トークン

await を含む処理には、ロールごとの roll_sequence_id または同等の世代番号を持たせる。

    var roll_sequence_id := 0

    func is_sequence_current(sequence_id: int) -> bool:
        return sequence_id == roll_sequence_id and is_inside_tree()

次のイベントでは世代番号を無効化し、await 後に必ず確認する。

- ロールキャンセル
- pauseからの復帰不能
- シーン終了
- MAP／ボス遷移による表示切り替え
- エラーによる中断

## 6. ロール結果の確定契約

### 6.1 論理値と表示値を分ける

ロール中は、次の値を混同しない。

| 値 | 用途 | 確定タイミング |
|---|---|---|
| preview_values | ROLLING中の見た目。まだ結果ではない | フレームまたは制御タイミングごとに更新可 |
| committed_values | 停止後にゲームへ渡す結果 | 該当ダイスの停止入力時に一度だけ |
| display_face | プレイヤーへ描画する面 | committed_values と同時に固定 |

停止入力後は、該当インデックスについて preview_values を更新してもよいが、描画には使わない。  
committed_values と display_face を唯一の表示ソースにする。

### 6.2 停止イベント

停止入力を受けた瞬間に、次を同じ処理単位で行う。

1. 現在の対象ダイスの値を committed_values[index] へコピーする。
2. locked_dice_count を1増やす。
3. display_face[index] を固定する。
4. 該当ダイスへの次の停止入力を無効にする。
5. MapDieBillboard.show_face() に固定値を渡す。
6. 全ダイスが固定されたら TurnPhase.SETTLING へ遷移する。

論理的なロール結果を GameState、ルート計算、報酬処理へ渡すのは、全ダイス固定後だけとする。  
結果処理を先に進めても、キャラクター移動は余韻シーケンス後に開始する。

## 7. サイコロ演出

### 7.1 変更対象

- scripts/game/map_dice_overlay.gd
- 必要に応じて scripts/game/dice_presentation_3d.gd
- 呼び出し側の scripts/app/main.gd

既存の3Dプレゼンターを無条件に削除せず、通常画面・マップ画面・QA用の責務を確認してから変更する。

### 7.2 投射パラメータ

投射開始時に、ダイスごとまたは投射単位で一度だけ決める。

    roll_visual_seed
    drift_direction
    drift_distance
    bounce_height
    primary_rotation_axis
    secondary_rotation_amount

初期値の範囲：

    上下バウンド        8〜12px
    横ドリフト総量     10〜16px
    方向転換            0〜1回
    主回転軸            1つ
    補助回転            主回転の15〜30%

禁止事項：

- 毎フレーム乱数で方向・速度・出目を変える
- 1フレームで10pxを超える位置ジャンプ
- トレイ外へ飛び出す
- カメラをダイスへ追従させる
- 停止後に別の出目面を表示する

目押しの学習可能性を残すため、乱数は揺らぎのためだけに使い、動きの構造は安定させる。

### 7.3 台座とダイス本体

- 台形の側面ポリゴンがある場合は削除する
- ダイス本体はトレイ面に近い基準位置へ配置する
- 側面削除後に、サイズ・枠線・ピップのコントラストが不足しないか確認する
- 立体感は台座ではなく、影・接地タイミング・小さなバウンスで補う

### 7.4 楕円影

影はダイスと同じ回転変換を受けない親座標で描画する。  
影の中心はダイスの水平移動には追従してよいが、ダイスの回転には追従させない。

推奨初期値：

    接地時の幅        34〜42px
    接地時の高さ        8〜12px
    最大跳躍時の幅     24〜30px
    最大跳躍時の高さ     5〜8px
    接地時alpha         0.28〜0.38
    最大跳躍時alpha     0.10〜0.18

Godotに楕円描画APIがない場合は、非等方スケールしたテクスチャ、またはポリゴン近似を使う。  
影の幅・高さ・alphaは bounce_y から補間し、跳躍中は小さく薄く、接地時は大きく濃くする。

### 7.5 SETTLING

停止入力または最後の自動停止で、ただちに SETTLING を開始する。

- 表示面と論理値は固定
- 高さを0へ戻す
- 横ドリフトを中心へ収束
- 影を濃く・大きくする
- 最後に1回だけ小さな反発を入れる
- 最後の数フレームで回転を0へ収束する

SETTLING は初期値140ms、許容範囲120〜180msとする。  
見た目のバウンドを追加しても、この時間内に収める。

## 8. 停止から移動までの余韻

### 8.1 初期値

    SETTLING       140ms
    RESULT_LOCK    200ms
    SLOT_TRANSFER  140ms
    MOVE_PREP      100ms
    合計           580ms

許容範囲は合計500〜650ms。  
端末性能によるフレーム落ちは別として、演出時間そのものが範囲外にならないこと。

### 8.2 フェーズの目的

#### RESULT_LOCK

- 出目の数字・ピップを最も読みやすい状態にする
- 着地点または移動距離を予告する
- 縁やトレイ枠の発光は1回だけ、弱く行う
- 大きなズーム、連続点滅、画面揺れは使わない

#### SLOT_TRANSFER

- 3ROLL SLOTまたは履歴枠に、確定した値が入ったことを見せる
- 該当枠の状態を1回だけ強調する
- 表示上の転写がないモードでは、空の移動演出を追加せず、枠の確定表示だけを使う
- 既存のPAIR / TRIPLE表示は、結果の意味を変更せず、転写後に接続する

#### MOVE_PREP

- キャラクターを3〜6px沈ませる
- 膝を曲げる、または短い溜めを入れる
- 影を一時的に約1.15倍へ広げる
- 顔や種別アイコンの可読性を落とさない

#### MOVING

- キャラクターは画面内のほぼ同じ位置で跳ねる
- ルートと背景を1マスずつ逆方向へ動かす
- 1マスあたり初期値156ms、許容範囲140〜200ms
- 次のマス種別を追える速度を保つ
- 移動数が0の場合は、MOVINGを経由せずマス解決へ進む

## 9. 通常マップ表示

### 9.1 8スロット

通常の直線ルートは、次の8スロットだけをタイルとして描画する。

    offset -1 : 直前マス
    offset  0 : 現在地
    offset +1 : 出目1
    offset +2 : 出目2
    offset +3 : 出目3
    offset +4 : 出目4
    offset +5 : 出目5
    offset +6 : 出目6

    const FIRST_OFFSET := -1
    const LAST_OFFSET := 6
    const SLOT_COUNT := LAST_OFFSET - FIRST_OFFSET + 1

SLOT_COUNT は配列長と一致させ、別の箇所で8を再定義しない。

### 9.2 遠方の道

offset >= 7 または offset <= -2 のマスは丸タイルとして描画しない。

代わりに次を描画する。

- 既存ルートポリライン
- 薄い道筋
- 背景へ溶ける線
- 必要な場合のみ小さな分岐方向

遠方マスの枠、影、絶対番号、相対番号、アイコンは表示しない。  
ポリライン自体は遠方まで残してよい。今回の制限対象はタイルUIであり、ルートデータや背景の道ではない。

### 9.3 タイルサイズ

現在のコードでは中心タイルがすでに約44pxで、遠方ほど小さくなる実装がある。  
したがって「34pxから一律1.3倍」と決め打ちせず、実際の tile_rects() の結果を基準に調整する。

初期目標：

    現在地の直径       44〜48px
    前方1〜3の最小径    40px以上
    前方4〜6の最小径    36px以上

360×640で8スロット、相対ラベル、ピクトグラム、キャラクター、トレイが干渉しない最大値を採用する。  
720×1280では単純な2倍拡大ではなく、既存のUIスケール規則に従う。

### 9.4 相対ラベル

| 位置 | 表示 |
|---|---|
| offset -1 | 番号なし |
| offset 0 | 現在地の絶対番号 |
| offset +1〜+6 | タイル上の相対出目 1〜6 |
| 全体MAP | 既存どおり絶対番号を表示してよい |

相対ラベルは小さな丸背景などでタイル本体から分離する。  
現在地の絶対番号と相対ラベルを同じ位置へ重ねない。

## 10. 分岐と円環

### 10.1 分岐

分岐位置を tourism_map_view.gd に固定値で持たせず、既存のルートモデルまたはステージデータから取得する。  
マス番号32は現行ステージのテスト例であり、仕様上の固定ルールにしない。

分岐点が現在地から前方6マス以内にある場合：

- 分岐までの共通部分は通常の8スロットに描画する
- 分岐後の本線と近道は、各先3マスまで補助表示する
- 本線を通常の濃さ、近道を少し薄い線と同程度の視認性で示す
- それぞれのマスに同じピクトグラム体系を使う
- 選択前は両方を比較可能にする
- 選択後は選択ルートを前方6マスへ展開し、非選択ルートは薄い線だけにする
- 残り移動数がある場合、選択後のルートで移動を継続する

分岐の補助タイルは「8スロットの通常表示を無制限に増やす」ものではない。  
通常の8スロットと補助表示が重なる場合は、前方の判断に必要な情報を優先する。

### 10.2 円環

円環・迷路・ループ系ルートでは、通常の8スロット表示を使わない。

- 円環全体を表示する
- 現在地を強調する
- EXITを最優先で表示する
- EXITまでの必要出目を EXIT 4 のように大きく表示する
- 各マス種別を識別可能にする
- 下部トレイ、ROLLボタン、主要操作位置は変更しない

円環中は前方マスの 1〜6 よりEXIT距離を優先する。  
V06PlaySession またはルートデータの「円環である」情報を使い、画面名や特定のルート文字列だけに依存しない。

## 11. マス種別の共通ビジュアル仕様

### 11.1 3層識別

すべてのマスは次の3層で識別する。

1. 外形
2. 中央ピクトグラム
3. 補助色

色をグレースケール化しても、RISKとREST、特殊マスとNORMALの大分類が分かること。

### 11.2 推奨マッピング

| 種別 | 外形 | 中央ピクトグラム | 補助色 |
|---|---|---|---|

### 11.3 描画ルール

- 中央ピクトグラムはタイル面積の55〜65%
- 線幅は360×640で潰れない太さにする
- 1タイルにつき中央記号は1つ
- 文字による ITEM、EVENT、漢字、店名などを中央アイコンにしない
- 昼夜で記号体系を変えず、夜景では輪郭または内側プレートで補正する
- 細かい装飾、細線、複雑な文字は省略する
- NORMALを無情報にせず、足跡または足場として認識できる弱い記号を残す
